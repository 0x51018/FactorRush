# 현재 작업 계획

이 문서는 형진 님과 합의된 다음 구현 범위를 기준으로, 이후 작업 턴에서 참조할 고정 계획입니다.
범위가 바뀌면 이 문서와 진행 플랜을 함께 갱신합니다.

## 확정된 범위

- 라이트 모드 포인트 색상은 더 붉게, 다크 모드는 조금만 붉게 보정한다.
- 랜딩 화면의 시작 모드 선택도 버튼형 선택 UI로 통일한다.
- 로비의 초대 링크 카드는 URL 가독성과 복사 UX를 개선한다.
- 로비 채팅은 카드형 말풍선보다 더 단순한 로그형 표현으로 정리한다.
- 로비의 규칙 카드 높이는 내용에 맞게 줄이고, 남는 공간은 채팅에 우선 배정한다.
- 인게임 진행 상태 배지는 라이트 모드에서도 글자가 잘 보이도록 대비를 조정한다.
- 룰북에는 정확한 점수 산식과 변수 설명을 명시한다.
- Decimal / Binary 모드에는 실시간 변환 프리뷰를 기본값 `켜짐`으로 제공하고, 설정에서 끌 수 있게 한다.
- Prime Factor 기본 모드에는 `서든데스 연장` 옵션을 추가한다.
  - 제한 시간이 끝날 때까지 아무도 정답을 못 맞췄을 경우, 이후 첫 정답자 1명만 점수를 얻는다.
- 골든벨 모드는 전체 라운드 제한 시간을 제거하고, 정답 외치기 성공자의 답변 제한 시간만 10초로 유지한다.
- 단, 전체 매치 시간이 1시간을 넘기면 자동 종료 안전장치를 둔다.
- 골든벨 입력 중 텍스트 실시간 공개는 이번 범위에서 제외한다.
- 중도 참가 시 관전자 입장 기능은 이번 로드맵에 포함한다.

## 철회되거나 제외된 제안

- 선착순 모드의 전체 시간 무제한은 적용하지 않는다.
- 골든벨 답변 중 입력 중인 텍스트를 다른 사람에게 실시간으로 보여주는 기능은 적용하지 않는다.

## 턴별 계획

### 턴 1

- 공통 UI 폴리시 정리
- 라이트/다크 포인트 색 보정
- 랜딩 시작 모드 버튼형 통일
- 초대 링크 카드/복사 UX 개선
- 로비 규칙-채팅 레이아웃 단순화
- 진행 상태 배지 대비 수정

### 턴 2

- 설정/룰북 고도화
- 정확한 점수 산식 명시
- Binary 실시간 변환 프리뷰 추가
- 프리뷰 on/off 설정 추가
- Prime Factor 세부 규칙 문구 재정리

### 턴 3

- 게임 규칙 확장
- Prime Factor `서든데스 연장` 옵션 추가
- 골든벨 전체 제한 제거
- 골든벨 답변권 10초 유지
- 1시간 매치 상한 안전장치 추가

### 턴 4

- 중도 참가 관전자 기능
- 플레이어/관전자 분리
- 입장 정책과 UI 표시 정의
- 상태 동기화 및 소켓 계약 추가

### 턴 5

- 회귀 테스트/수동 QA/배포 준비
- smoke/E2E 보강
- 규칙별 멀티플레이 검증
- 문서 정리

## 각 턴의 완료 기준

### 턴 1 완료 기준

- 랜딩/로비/인게임의 공통 톤과 버튼형 UI가 통일된다.
- 링크 복사 UX와 로비 레이아웃이 더 간결해진다.
- 라이트 모드 진행 배지 가독성 문제가 사라진다.

### 턴 2 완료 기준

- 룰북만 읽어도 점수 계산 방식과 Binary 보조 기능을 이해할 수 있다.
- Binary 입력 중 실시간 변환 프리뷰가 안정적으로 보인다.

### 턴 3 완료 기준

- Prime Factor와 골든벨 신규 옵션이 서버 authoritative 규칙으로 동작한다.
- 1시간 매치 상한이 실제 종료 조건으로 작동한다.

### 턴 4 완료 기준

- 진행 중인 방에 들어온 사용자는 관전자로 안전하게 합류할 수 있다.
- 관전자와 플레이어 상태가 UI와 서버에서 명확히 구분된다.

### 턴 5 완료 기준

- 자동 테스트와 수동 QA를 통과한다.
- 현재 운영 문서와 규칙 문서가 최신 상태로 정리된다.

## 현재 진행 상태 체크포인트

이 섹션은 작업을 잠시 멈췄다가 다시 이어갈 때, 정확히 어디서부터 재개해야 하는지 고정하기 위한 체크포인트입니다.

### 마지막으로 검증이 끝난 지점

- 턴 1 완료 및 검증 완료
- 턴 2 완료 및 검증 완료
- 턴 3 완료 및 검증 완료
- 턴 4 완료 및 검증 완료
- 마지막으로 확실히 통과한 검증은 아래와 같습니다.
  - `npm run build`
  - `SMOKE_SERVER_URL=http://127.0.0.1:3001 npm run smoke:dev`
  - `PLAYWRIGHT_BASE_URL=http://127.0.0.1:3000 npm run test:e2e`
  - 결과: `6 passed`

### 현재 중단 지점

- 턴 5의 UI 폴리시/룰북 분리 패스까지 반영된 상태입니다.
- 턴 1~4의 구현과 자동 검증은 유지되며, 턴 5도 최신 코드 기준으로 자동 검증을 다시 통과했습니다.
- 현재 남은 일은 형진 님의 수동 QA와, 필요 시 그 피드백을 반영한 마지막 폴리시 정리입니다.

### 이번 단계까지 완료된 핵심 변경

- Prime Factor 기본 모드용 `서든데스 연장` 옵션 구조 추가
- 골든벨 모드의 전체 라운드 제한 제거 구조 추가
- 골든벨 답변권 10초 상수 및 서버 로직 연결
- 전체 매치 1시간 상한 안전장치 구조 추가
- 룰북에서 점수 수식을 분리하고 KaTeX 기반 `점수 공식` 모달 추가
- 로비/인게임 채팅을 카드형 말풍선 대신 로그형 채팅으로 재정리
- 인게임 우측 레일을 `리더보드 + 고정 채팅 도크` 구조로 재배치
- 설정 모달 공통 항목 2열 배치 및 세부 옵션 2열 카드 그리드화
- 정답 공개 오버레이 상단 밀도 축소 및 점수 변화 보드 공간 확장
- 라운드 스냅샷에 `hasRoundTimer`, `isSuddenDeath` 추가
- 룸 스냅샷에 `matchEndsAt` 추가
- 룰북/설정/상태 문구에서 신규 규칙 표시용 문구 추가
- 진행 중 입장 시 관전자 분기 추가
- 로비에서 관전자의 플레이어 좌석 합류 기능 추가
- smoke/e2e에 관전자 시나리오 추가

### 재개 시 첫 순서

1. 형진 님의 수동 QA 피드백을 받는다.
2. 필요하면 마지막 폴리시 수정과 수동 캡처 검증을 추가한다.
3. PR 생성 전 마지막 `build/smoke/e2e`를 한 번 더 돌린다.
4. 이후 PR 또는 배포 단계로 이동한다.

### 재개 전 환경 메모

- 현재 개발 검증은 `3000/3001` 기준으로 다시 고정해 둔 상태를 기준으로 한다.
- 만약 다음 재개 시 포트 충돌이 보이면, 먼저 기존 `node` dev 프로세스를 정리하고 시작한다.
